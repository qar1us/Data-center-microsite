# Images

Drop site photography and graphics here. Claude will reference these from the pages.

## Naming
Use lowercase, hyphenated, descriptive names so it's obvious where each goes:

- `hero.jpg` — main hero background (data center exterior / aerial / transmission lines)
- `role-resident.jpg`, `role-county.jpg`, `role-state.jpg`, `role-cso.jpg`, `role-federal.jpg`
- `win-port-washington.jpg`, `win-cascade-locks.jpg`, `win-pima.jpg`, `win-minnesota.jpg`
- `why-bills.jpg`, `why-grid.jpg`, `why-neighborhood.jpg`
- `divider-1.jpg`, etc. for section breaks

(Names are just suggestions — anything clear works. Tell me which file goes where if it's not obvious.)

## Specs
- **Format:** JPG or WebP for photos (smaller); PNG/SVG for logos/graphics.
- **Hero / full-width:** ~2400px wide, landscape. Keep important content away from the left third (text sits there).
- **Cards / thumbnails:** ~1000–1200px wide.
- **File size:** aim < 400 KB each (compress before adding; e.g., squoosh.app). The whole page should stay light.
- **Orientation:** mostly landscape; portrait is fine for specific cards.

## Treatment
Photos will be brand-treated in CSS (e.g., duotone or a dark scrim), so they don't need to be pre-edited — drop in the raw photo and it will be unified automatically. If you'd rather keep a photo full-color, just say so.

## Licensing  ⚠️
Only add images we have rights to use:
- Free: Unsplash, Pexels, Wikimedia Commons (check each license).
- Paid stock (with a license), or ARI's own/commissioned photos.
- Avoid grabbing random images from Google — many are copyrighted.
